## Configuration

### getConfig()

Internal function that retrieves and validates required GalioPay environment variables.

**Environment Variables Required:**

- `GALIOPAY_API_BASE` - Base URL for the GalioPay API
- `GALIOPAY_API_KEY` - API key for authentication
- `GALIOPAY_CLIENT_ID` - Client identifier for API requests

**Error Handling:**

Throws an error if any required environment variable is missing:
```typescript
throw new Error("Missing required GalioPay environment variables");
```

## Payment Links

### createPaymentLink()

Creates a new payment link for collecting payments from customers.

**Function Signature:**
```typescript
function createPaymentLink(
  payload: CreatePaymentLinkInput
): Promise<CreatePaymentLinkResponse>
```

**Parameters:**

<ParamField path="payload" type="CreatePaymentLinkInput" required>
  Payment link configuration object
  
  <Expandable title="properties">
    <ParamField path="items" type="GalioPayItem[]" required>
      Array of items to be paid for
    </ParamField>
    
    <ParamField path="referenceId" type="string" required>
      Unique reference identifier for this payment link
    </ParamField>
    
    <ParamField path="description" type="string">
      Optional description of the payment
    </ParamField>
    
    <ParamField path="establishmentName" type="string">
      Name of the establishment receiving payment
    </ParamField>
    
    <ParamField path="sellerName" type="string">
      Name of the seller
    </ParamField>
    
    <ParamField path="backUrl" type="object">
      Redirect URLs after payment completion
      
      <Expandable title="properties">
        <ParamField path="success" type="string">
          URL to redirect to on successful payment
        </ParamField>
        
        <ParamField path="failure" type="string">
          URL to redirect to on failed payment
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

**Returns:**

<ResponseField name="url" type="string" required>
  The generated payment link URL that can be shared with customers
</ResponseField>

**Example:**

```typescript
const response = await createPaymentLink({
  items: [
    {
      title: "Premium Subscription",
      quantity: 1,
      unitPrice: 2999,
      currencyId: "ARS",
      imageUrl: "https://example.com/subscription.png"
    }
  ],
  referenceId: "order-12345",
  description: "Monthly premium subscription",
  establishmentName: "My SaaS Company",
  backUrl: {
    success: "https://myapp.com/payment/success",
    failure: "https://myapp.com/payment/failure"
  }
});

console.log(response.url); // Payment link URL
```

**API Endpoint:** `POST /payment-links`

### getPaymentLink()

Retrieves details of a specific payment link using its ID and proof token.

**Function Signature:**
```typescript
function getPaymentLink(
  id: string,
  proof: string
): Promise<PaymentLinkDetails>
```

**Parameters:**

<ParamField path="id" type="string" required>
  The payment link ID
</ParamField>

<ParamField path="proof" type="string" required>
  Proof token for verification (passed as query parameter)
</ParamField>

**Returns:**

<ResponseField name="id" type="string" required>
  Payment link identifier
</ResponseField>

<ResponseField name="proofToken" type="string" required>
  Verification proof token
</ResponseField>

<ResponseField name="items" type="GalioPayItem[]" required>
  Array of items in the payment link
</ResponseField>

<ResponseField name="referenceId" type="string" required>
  Reference identifier for this payment link
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status of the payment link
</ResponseField>

<ResponseField name="paymentId" type="string | null" required>
  Associated payment ID (null if not yet paid)
</ResponseField>

**Example:**

```typescript
const linkDetails = await getPaymentLink(
  "pl_abc123",
  "proof_xyz789"
);

console.log(linkDetails.status); // "pending" | "completed" | etc.
console.log(linkDetails.paymentId); // "pay_def456" or null
```

**API Endpoint:** `GET /payment-links/{id}?proof={proof}`

**Note:** This endpoint does not require authentication (`withAuth = false`)

```typescript
function getPayment(id: string): Promise<PaymentDetails>
```

**Parameters:**

</ResponseField>

```typescript
console.log(payment.netAmount); // 2849 (after fees)
```

**API Endpoint:** `GET /payments/{id}`

### refundPayment()

Processes a refund for an existing payment.

**Function Signature:**
```typescript
function refundPayment(
  id: string,
  payload: RefundPaymentInput
): Promise<RefundPaymentResponse>
```

**Parameters:**

<ParamField path="id" type="string" required>
  The payment ID to refund
</ParamField>

<ParamField path="payload" type="RefundPaymentInput" required>
  Refund configuration
  
  <Expandable title="properties">
    <ParamField path="reason" type="string">
      Reason for the refund
    </ParamField>
    
    <ParamField path="refundType" type="'total' | 'partial'">
      Type of refund (defaults to "total" if not specified)
    </ParamField>
    
    <ParamField path="refundAmount" type="number">
      Amount to refund (required when refundType is "partial")
    </ParamField>
  </Expandable>
</ParamField>

**Returns:**

<ResponseField name="success" type="boolean" required>
  Whether the refund was successful
</ResponseField>

<ResponseField name="message" type="string" required>
  Descriptive message about the refund result
</ResponseField>

<ResponseField name="payment" type="object">
  Updated payment information
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Payment identifier
    </ResponseField>
    
    <ResponseField name="status" type="string">
      New payment status after refund
    </ResponseField>
  </Expandable>
</ResponseField>

**Example (Total Refund):**

```typescript
const result = await refundPayment("pay_def456", {
  reason: "Customer requested cancellation",
  refundType: "total"
});

console.log(result.success); // true
console.log(result.message); // "Refund processed successfully"
console.log(result.payment?.status); // "refunded"
```

**Example (Partial Refund):**

```typescript
const result = await refundPayment("pay_def456", {
  reason: "Partial order cancellation",
  refundType: "partial",
  refundAmount: 1000
});

console.log(result.success); // true
```

**API Endpoint:** `POST /payments/{id}/refund`

## Error Handling

All client functions implement consistent error handling:

**Configuration Errors:**
```typescript
// Thrown when environment variables are missing
throw new Error("Missing required GalioPay environment variables");
```

**API Errors:**
```typescript
// Thrown when API request fails
// Error message is extracted from response.error if available
// Otherwise falls back to generic message
throw new Error(getErrorMessage(data, "GalioPay request failed"));
```

**Error Handling Pattern:**

```typescript
try {
  const response = await createPaymentLink({
    items: [...],
    referenceId: "order-123"
  });
  // Handle success
} catch (error) {
  if (error instanceof Error) {
    console.error("Payment link creation failed:", error.message);
  }
  // Handle error
}
```

**Implementation Details:**

The `requestGalioPay()` internal function (`/home/daytona/workspace/source/src/lib/galiopay/client.ts:36-66`):
- Automatically adds authentication headers (Bearer token + x-client-id)
- Sets `Content-Type: application/json`
- Disables caching with `cache: "no-store"`
- Parses JSON responses
- Throws descriptive errors on non-OK responses
